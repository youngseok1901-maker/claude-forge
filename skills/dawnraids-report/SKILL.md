# Dawnraids Report (현장조사 일일보고)

`dawnraids_report` 서브에이전트를 호출해 현장조사 카카오톡 기록을 BKL 양식 보고서(.docx)로 변환하고 구글 드라이브에 업로드한다.

## 동작

`$ARGUMENTS`를 그대로 `dawnraids_report` 에이전트에 전달하여 **Agent 도구로 디스패치**한다.

- ARG1: 카카오톡 내보내기 파일 경로(.txt/.rtf, 로컬) — 또는 구글 드라이브 파일명/ID
- ARG2(선택): 사건명 (예: "삼화페인트")
- ARG3(선택): N일차 (예: "1일차")

인자가 비어 있으면 사용자에게 ① 카카오톡 파일 경로 ② 사건명 ③ 몇 일차인지를 먼저 묻고,
받은 값을 인자로 구성해 에이전트를 호출한다.

## 에이전트가 수행하는 일 (요약)

1. 양식 파일 + few-shot 샘플 정독 (`~/.claude/agent-memory/dawnraids_report/samples/`)
2. 카카오톡 원본 취득(.rtf는 `textutil`로 변환)
3. (요청자료 한정) 전일 보고서 참고 — I. 개요는 변호사 직접 작성 영역이라 승계하지 않음
4. 피조사자·인터뷰/진술 Q&A·요청자료 추출, 잡담 필터링
5. **검토 게이트**: 초안 제시 → 사용자 "승인" 후에만 진행
6. `.docx` 생성(I.개요·II.대응방안 플레이스홀더 + III 자료열람/진술조사/요청자료/향후일정)
   → 구글 드라이브 `92. 현장조사 보고` 업로드
7. webViewLink 반환 + run_log 기록

## 주의

- 변호사-의뢰인 특권 기밀 자료다. 내용을 외부 웹서비스로 전송하지 않는다.
- I. 개요와 II장 대응방안은 변호사가 직접 작성한다 — 에이전트는 플레이스홀더만 둔다.
- 피조사자·조사관 이름은 실명 기재, 전화번호·이메일 등 연락처만 마스킹한다.
- 양식을 바꾸려면 에이전트 코드가 아니라
  `~/.claude/agent-memory/dawnraids_report/samples/00_보고서_양식.md` 만 수정한다.
